# FAQ

## 手机端无法播放首页音频（Splash 音效）？

**原因：** 现代浏览器（尤其 iOS Safari、部分 Android）的**自动播放策略**要求：音频必须在**用户主动操作**（如点击、触摸）之后才能播放，且 `AudioContext` 在移动端默认处于 `suspended` 状态，需在用户手势中调用 `resume()` 并等待其完成后再输出声音。

**当前实现：**

- 首页进入时的鸟群/环境音**不会自动播放**，需用户**点击右上角喇叭图标**才会开启。
- 代码已做兼容：
  - 在用户点击/触摸喇叭时创建 `AudioContext` 并调用 `audioCtx.resume()`，用 **Promise** 等待恢复后再拉高音量，避免移动端“点了也没声”。
  - 为喇叭按钮绑定了 **`touchend`**（并 `preventDefault`），保证在触屏上也能算作用户手势，且不触发二次点击。

**使用方式：**

1. 打开首页（Splash 画面）。
2. **点击或触摸一次右上角喇叭图标**（静音图标）。
3. 图标变为“扬声器”且应能听到环境音；若仍无声音，请检查设备静音/媒体音量及浏览器权限。

**若仍无法播放：**

- 确认设备未处于静音、媒体音量已打开。
- iOS：设置 → Safari → 媒体与设备访问等权限未限制。
- 部分浏览器在“省电/低数据”模式下会限制 Web Audio，可尝试关闭或换用 Chrome/Safari 最新版再试。
